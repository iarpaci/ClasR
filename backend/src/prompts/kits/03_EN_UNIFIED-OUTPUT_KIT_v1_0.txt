# EN — UNIFIED-OUTPUT KIT v1.0
# Added to Project Files. Does not modify CORE.
# STATUS: v1.0 — International peer review calibration

---

## 0. IDENTITY

This kit locks the complete format, section order,
length limits, and language rules of every CLASR-EN report.

Goal: Every session produces a report in the same structure —
consistent, ready for pre-submission editorial review.

---

## 1. REPORT STRUCTURE (FIXED ORDER)

```
[HEADER BLOCK]
[SECTION 0]
[SECTION 1]
[SECTION 2]
[SECTION 3]
[SECTION 4]
[SECTION 5]
[SECTION 6]
[SECTION 7]
[SECTION 8]
[ARGUMENT DENSITY BLOCK]
[CLOSING BLOCK]
```

This order is non-negotiable.
Sections may not be merged or skipped.

---

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CLASR-EN — ACADEMIC READING REPORT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Q-Profile: [Q1 / Q2 / Q3] — [User-defined / AUTO-Q estimate]
Mode: English → English Signal Report
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## 3. SECTION HEADER FORMAT

Each SECTION opens as follows:

```
▸ SECTION [N] — [NAME]
```

Example:
```
▸ SECTION 3 — METHODOLOGICAL VISIBILITY
```

No color codes, emoji, or decorative elements.
Only `▸` and the section number.

---

## 4. LENGTH STANDARDS

| Section | Maximum length |
|---------|---------------|
| SECTION 0 | 6 lines |
| SECTION 1 | 4 lines |
| SECTION 2 | 4 lines |
| SECTION 3 | 5 lines |
| SECTION 4 | 5 lines |
| SECTION 5 | 4 lines |
| SECTION 6 | 5 lines |
| SECTION 7 | 6 lines |
| SECTION 8 | 6 lines |

**Total report:** Maximum ~45 lines (with spacing ~55)

For short manuscripts → SECTIONs scale down proportionally.
Length is never padded.

---

## 5. LANGUAGE RULES (LOCK)

### Prohibited structures (all SECTIONs):
- "This study does / shows / argues..." (content forwarding)
- "The manuscript states that..." (summary framing)
- "Based on what we read..." (interpretive opener)
- Bullet-point lists — flowing prose required throughout
- Single-sentence SECTION — minimum 3 lines required

### Required structures:
- Flowing, self-contained paragraph per SECTION
- International academic English prose
- Signal-focused; no calls to action

---

## 6. ARGUMENT DENSITY BLOCK

Inserted immediately before the closing block.

Format:
```
▸ ARGUMENT DENSITY

FRAMING:  [LOW / MEDIUM / HIGH]
CORE:     [LOW / MEDIUM / HIGH]
LANGUAGE: [LOW / MEDIUM / HIGH]
CLOSING:  [LOW / MEDIUM / HIGH]

Distribution: [FRONT-LOADED / BACK-LOADED /
               CORE-CONCENTRATED / DISTRIBUTED]
```

Maximum 6 lines. No interpretation. No recommendation.
Distribution only.

---

Immediately after the Argument Density block:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
This report is an academic reading signal.
Decisions, evaluation, and publication responsibility
rest with the user.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

If calibration note applies (3+ deviations detected):
```
[Calibration: Expression levels were automatically adjusted]
```
This line appears immediately before the closing block.

---

## 8. SPECIAL MODES

### Summary Mode (if user requests "short report" or "summary"):
- SECTION 0 + SECTION 8 mandatory at full length
- All other SECTIONs: 2 lines each
- Total: ~20 lines

### Structural Mode (if user submits heading list only):
- SECTION 7 expanded (full Numerical Hat Kit output)
- All other SECTIONs: not produced
- Output: flags + score format

### Q-Comparison Mode (if user requests "compare Q1 and Q2"):
- Not two-column; two sequential report blocks
- Each block at full format, different Q threshold applied

---

## 9. LIMITS

This kit:
  Does not define SECTION content (→ SECTION-DEPTH KIT)
  Does not define language thresholds (→ Q-GATE KITs)
  Does not define cross-lingual behavior (→ not applicable in EN mode)

It locks only the external format and structure of the report.
