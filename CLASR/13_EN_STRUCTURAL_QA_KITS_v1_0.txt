# EN — NUMERICAL HAT KIT v1.0
# (Quantitative Academic Heading Review System)
# Added to Project Files. Does not modify CORE.
# STATUS: v1.0 — Fully automatic, content-free, structure-only

---

## 0. IDENTITY

Type: STRUCTURAL QA KIT
Operates via: Section headings only
No content reading · No rewriting · No recommendations
Fully automatic — no user configuration

---

## 1. INPUT CONTRACT

Input: Section headings list (Table of Contents or extracted headings)
Language: English or mixed (auto-detected)
No metadata, no field selection, no journal specification required.
Output: Heading map + structural flags + structural scores

---

## 2. AUTOMATIC HEADING CLASSIFICATION

All headings are mapped to fixed structural classes.

**FRONT** — Identity and front matter
**ABSTRACT** — Abstract layer
**INTRO** — Introduction and framing
**DATA** — Data, sample, variables
**METHODS** — Method, model, design
**DOMAIN_METHODS** — Spatial, temporal, field-specific methods
**REPRO** — Software, code, reproducibility
**RESULTS** — Findings
**DISCUSSION** — Discussion and interpretation
**CONCLUSION** — Closing
**ETHICS** — Ethics, declarations, transparency
**BACK** — References and appendices
**SUBMISSION_ONLY** — Cover letter, reviewer response (excluded from scoring)

---

## 3. AUTOMATIC STUDY-TYPE DETECTION

Study type inferred from heading signals:

"GIS / Spatial / Resolution" → Spatial analysis
"Panel / DiD / IV" → Econometric causal study
"Bayesian / Prior" → Bayesian study
"Cross-validation / Hyperparameters" → ML / simulation
"PRISMA / Meta-analysis" → Systematic review
"Ethics / Consent / Clinical" → Clinical / human subjects

Multiple types may co-exist. User is not asked.

---

## 4. MANDATORY SECTION SET (AUTO-BUILT)

Base mandatory expectations (quantitative default):
- Abstract
- Introduction
- Data (or Sample / Study Area)
- Methods / Model
- Results
- Discussion
- Limitations
- Conclusion
- References

Conditional (auto-triggered by detected type):
- Spatial study → projection / resolution block
- Causal study → identification strategy
- ML / simulation → validation + reproducibility
- Human subjects → ethics + consent
- Meta-analysis → reporting checklist signals

---

## 5. DEPENDENCY RULES

- Results require Methods
- Methods require Data
- Robustness / Sensitivity require Main Results
- Discussion requires Results
- Generalizability requires Limitations
- Causal methods require Identification strategy
- Reproducibility claims require Code / Data availability

Violations generate flags only.

---

## 6. ORDER REFERENCE (SOFT)

Preferred reference flow:
FRONT → INTRO → DATA → METHODS → RESULTS → DISCUSSION → CONCLUSION → ETHICS → REFERENCES

Deviations generate warnings, not errors.

---

## 7. FLAG SYSTEM

- MANDATORY_MISSING
- CONDITIONAL_MISSING
- DEPENDENCY_BREAK
- ORDER_WARNING
- DOMAIN_SPEC_GAP
- AMBIGUOUS_HEADING
- REDUNDANT_SECTION
- SUBMISSION_ONLY_IGNORED

---

## 8. OUTPUT CONTRACT

System outputs only:
- Heading → structural class map
- Structural flags
- Structural scores (coverage, consistency, transparency)

No content. No recommendations.

---

## 9. INTEGRATION WITH CLASR-EN

Numerical Hat Kit flags are reported within SECTION 7 — Structural Integrity.
In Structural Mode (heading list only input):
  SECTION 7 expands to full Numerical Hat output
  All other SECTIONs suppressed

---

## 10. CANONICAL DEFINITION

"This kit automatically reviews English quantitative academic manuscripts
using section headings alone, infers study type and structural expectations,
and flags structural gaps without generating or evaluating content."

---

# NUMERICAL HAT KIT v1.0 — END

---

## 0. SYSTEM IDENTITY

This file defines a unified meta-system hosting multiple
content-free, heading-based academic QA products
under a single automatic engine.

Core principle:
  No content reading
  No rewriting
  No recommendations
  No user configuration
  All products operate via structural signals only

---

## 1. INPUT CONTRACT

Input: Section headings list
Language: English (auto-detected)
Writing regime: Quantitative / empirical (auto-detected)
No metadata or profile selection.

---

## 2. CORE ENGINES

E1: English – Quantitative Structure Engine
E2: English – Theoretical Structure Engine

Multiple engines may run in parallel if signals overlap.

---

## 3. AUTO-ACTIVATED PRODUCT MODULES

### 3.1 Academic Argument Structure Checker
Detects presence / absence of:
Problem → Motivation → Claim → Support → Implication

Flags:
- ARGUMENT_GAP
- CLAIM_WITHOUT_SUPPORT
- IMPLICATION_WITHOUT_ANALYSIS

### 3.2 Thesis vs Article Structure Differentiator
Flags:
- THESIS_STYLE_IN_ARTICLE
- ARTICLE_STYLE_IN_THESIS
- STRUCTURAL_LENGTH_MISMATCH

### 3.3 Reviewer Expectation Mismatch Detector
Flags:
- CONTRIBUTION_UNCLEAR
- METHODS_UNDEREXPLAINED
- DISCUSSION_TOO_THIN

### 3.4 Redundancy & Over-Sectioning Checker
Flags:
- REDUNDANT_SECTION
- OVER_SECTIONING
- FUNCTIONAL_DUPLICATION

### 3.5 Discipline Signal Mismatch Detector
Flags:
- DISCIPLINE_STYLE_MISMATCH
- METHOD_DISCIPLINE_TENSION

### 3.6 Academic Maturity Level Estimator (non-evaluative)
Levels:
- EARLY_STAGE_PATTERN
- INTERMEDIATE_PATTERN
- ADVANCED_PATTERN

### 3.7 Pre-Review Desk-Reject Risk Scanner
Flags:
- DESK_REJECT_RISK_HIGH
- DESK_REJECT_RISK_MEDIUM

### 3.8 Template Drift Detector
Flags:
- TEMPLATE_DRIFT
- STRUCTURAL_INCONSISTENCY

---

## 4. FLAG AGGREGATION

All flags from engines and modules are aggregated.
No prioritization or recommendation is produced.
Flags are descriptive only.

---

## 5. OUTPUT CONTRACT

System outputs only:
- Heading → structural class map
- Activated engines / modules list
- Flags (grouped by module)
- Structural scores (coverage / consistency / coherence)

No advice. No scoring explanation. No text output.

---

## 6. CANONICAL STATEMENT

"This system automatically analyzes academic manuscripts using section
headings alone, runs multiple content-free structural QA engines in
parallel, and flags structural risks, gaps, and mismatches without
generating, rewriting, or evaluating content."
